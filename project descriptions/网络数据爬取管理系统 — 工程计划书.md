## 二、数据库设计

### 2.1 MySQL — 结构化数据（3NF）

存储系统中结构固定的实体信息。

**实体及关键字段：**

```
Website（网站）
  - id, domain, organization, contact, created_at

Webpage（网页）
  - id, url, website_id(FK), crawl_time, status

DataSource（数据源信息）
  - id, webpage_id(FK), publisher, publish_time, source_url

CrawlTask（爬取任务）
  - id, target_url, status, created_at, finished_at, created_by
```

**关系说明：**

- Website 1 → N Webpage（一个网站包含多个网页）
- Webpage 1 → N DataSource（一个网页对应多条数据源记录）
- CrawlTask 记录每次爬取操作的元信息

### 2.2 MongoDB — 非结构化内容

存储爬取到的正文、图片等结构不固定的内容。

**Collection 设计：**

```json
// contents（正文内容）
{
  "_id": ObjectId,
  "webpage_url": "https://...",
  "webpage_id": 123,
  "text_content": "...",
  "keywords": ["AI", "数据库"],
  "crawl_time": "2026-04-01T12:00:00"
}

// images（图片）
{
  "_id": ObjectId,
  "webpage_url": "https://...",
  "webpage_id": 123,
  "image_url": "https://.../pic.jpg",
  "description": "示意图",
  "crawl_time": "2026-04-01T12:00:00"
}
```

### 2.3 ER 图（简化）

```
Website ──< Webpage ──< DataSource
                │
          ┌─────┴──────┐
       Content       Image
      (MongoDB)    (MongoDB)
```

## 三、系统架构

```
┌─────────────────────────────────────┐
│           Vue 3 前端                 │
│       (Element Plus 组件)            │
└──────────────┬──────────────────────┘
               │ HTTP / REST
┌──────────────▼──────────────────────┐
│         FastAPI 后端                 │
│   路由 / 业务逻辑 / BackgroundTasks   │
└──────┬───────────────┬──────────────┘
       │               │
┌──────▼──────┐  ┌──────▼──────┐
│    MySQL    │  │   MongoDB   │
│  结构化数据  │  │  爬取内容   │
└─────────────┘  └──────▼──────┘
                        │
               ┌────────┴────────┐
               │  Scrapy Spider  │
               │  （后台执行）    │
               └─────────────────┘
```

**爬取流程：**

1. 用户在前端提交爬取目标 URL
2. FastAPI 接收请求，在 MySQL 中创建 CrawlTask 记录，立即返回任务 ID
3. FastAPI 通过 `BackgroundTasks` 在后台调用 Scrapy
4. Scrapy 抓取完成后，结构化数据写入 MySQL，内容数据写入 MongoDB，更新任务状态
5. 前端每5秒轮询任务状态接口，完成后刷新数据列表

## 四、功能模块

### 4.1 爬虫模块（成员 B）

- [ ] 实现基础 Spider，抓取目标网站的文本内容和图片 URL
- [ ] Scrapy Pipeline：文本内容写入 MongoDB，网页元信息写入 MySQL
- [ ] 支持深度限制（`DEPTH_LIMIT`）和请求间隔（`DOWNLOAD_DELAY`）配置，防止被封
- [ ] 爬取完成后更新 CrawlTask 状态

**Spider 入口示例：**

```python
# 由 FastAPI BackgroundTasks 触发
def run_spider(task_id: int, url: str):
    process = CrawlerProcess(get_project_settings())
    process.crawl(GeneralSpider, start_url=url, task_id=task_id)
    process.start()
```

### 4.2 后端 API（成员 A）

| 方法   | 路径                   | 说明                           |
| ------ | ---------------------- | ------------------------------ |
| POST   | `/api/tasks`           | 提交爬取任务                   |
| GET    | `/api/tasks`           | 查询任务列表及状态             |
| GET    | `/api/websites`        | 网站列表                       |
| GET    | `/api/webpages`        | 网页列表，支持按网站筛选       |
| GET    | `/api/contents/search` | 内容检索（关键字/时间/发布者） |
| GET    | `/api/images`          | 图片列表，支持按关键字筛选     |
| DELETE | `/api/webpages/{id}`   | 删除网页及其内容               |
| GET    | `/api/stats`           | 爬取统计数据                   |

### 4.3 前端界面（成员 B）

**页面列表：**

- **首页 / 仪表盘**：展示爬取任务总数、网站数、内容数、图片数的统计卡片
- **爬取管理**：输入目标 URL，提交任务，实时查看任务进度列表
- **内容检索**：关键字搜索 + 时间范围筛选 + 发布者筛选，结果分页展示
- **图片管理**：图片缩略图瀑布流展示，支持按描述关键字筛选
- **网站 / 网页管理**：查看已爬取的网站列表，支持删除操作

### 4.4 数据管理（成员 A）

- [ ] 网页及其关联内容、图片的级联删除
- [ ] 支持导出检索结果为 CSV（FastAPI StreamingResponse）
- [ ] 爬取统计：按网站、按日期统计数量

## 六、Docker Compose 配置

```yaml
# docker-compose.yml（结构示意）
services:
  mysql:
    image: mysql:8.0
    environment:
      MYSQL_DATABASE: crawler_db
      MYSQL_ROOT_PASSWORD: root123
    volumes:
      - ./mysql/init.sql:/docker-entrypoint-initdb.d/init.sql

  mongo:
    image: mongo:7.0

  backend:
    build: ./backend
    ports:
      - "8000:8000"
    depends_on:
      - mysql
      - mongo

  frontend:
    build: ./frontend
    ports:
      - "5173:5173"
    depends_on:
      - backend
```

启动整个系统只需：

```bash
docker compose up --build
```

## 九、风险与应对

| 风险                        | 可能性 | 应对方案                                                     |
| --------------------------- | ------ | ------------------------------------------------------------ |
| 目标网站反爬，Scrapy 被封   | 中     | 设置 `DOWNLOAD_DELAY=2`，配置随机 User-Agent，准备2–3个备用测试网站 |
| 前后端联调接口不一致        | 中     | 开发初期先用 FastAPI 自动生成的 Swagger 文档（`/docs`）对齐接口 |
| MongoDB 和 MySQL 数据不同步 | 低     | 以 MySQL 的 `webpage_id` 作为两库的关联键，删除时先删 MongoDB 再删 MySQL |
| 两人进度不同步              | 中     | 每周至少一次代码合并，使用 Git 分支开发，主分支保持可运行状态 |
| Docker 环境本地跑不通       | 低     | 先本地裸跑通，最后一周再容器化，期中演示可直接用本地环境     |
